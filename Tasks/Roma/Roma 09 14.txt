ОСНОВНЫЕ ЗАДАЧИ
- Рефакторинг клиентской части. Переименовать VariableType в VariableScope, а также значения перечисления
- Рефакторинг серверной части
- Во всех функциях автоматизации должны быть описания
- Валидация всех функций
- Инвалидация всех функций
- Демонстрация работы всех функций
- Окна выбора объектов. Должен работать выбор строки по двойному щелчку. Должны быть радиобаттоны выбора с возможносьтью отмены выбора
+ Должны работать все типовые сценарии управления
+ Запуск сценариев по событию в журнале
+ Процедура на планах должна иметь белый значек, а не цветной
- Убрать функцию Выборочный досмотр персонала. Заменить ее функцией получения случайного значения
- Перечисление Локальная переменная должно идти перед перечислением Глобальная переменная
- Редактор глобальной переменной - XDataGrid
- Функция задания значения переменной не позволяет выбрать объектную ссылку
- Функция показа окна с сообщением должна иметь настройку показа в той ОЗ, из которой была вызвана функция


ТИПОВЫЕ СЦЕНАРИИ УПРАВЛЕНИЯ
- Включение РМ при сработке АМ по событию (в журнале)
- Периодическое включение/выключение РМ (по расписанию)
- Сброс пожара во всех зонах, находящихся в пожаре (по кнопке)

ДОПОЛНИТЕЛЬНЫЕ ЗАДАЧИ
- Запуск сценариев по состоянию объекта
- Функции управления пользовательским интерфейсом
- Разбивка сценариев на выполняемые на сервере и выполняемые на клиенте
- Таймаут выполнения процедуры
- Передача в вызываемую процедуру аргументов в качестве локальных или глобальных переменных, а не только явных значений
- Агрументы типа список
